Language Guide

Devabhāṣā source is written in Devanagari and compiles to readable JavaScript. Statements end with a danda । (or ;), comments begin with #, and numbers may be Devanagari (०१२…) or ASCII.

How it works

The pipeline is a classic three stages — lexer → parser → codegen — plus a keyword layer that is the only Sanskrit-aware part. Identifiers you write in Devanagari are transliterated to stable ASCII in the output (गणना → gannanaa), so the emitted JavaScript is portable and debuggable. Nothing is interpreted at runtime — Devabhāṣā is a pure transpiler.

Variables & functions

नियत is a constant, चर is reassignable. कार्य is a function — as a statement or an expression. फलम् returns.

नियत पाई = 3.14159।           # const
चर गणना = ०।                   # let (reassignable)

कार्य वर्गः (सङ्ख्या) {           # function squares its argument
    फलम् सङ्ख्या * सङ्ख्या।
}

नियत द्विगुण = कार्य(न){ फलम् न * २। }।   # function expression

Control flow

यदि (गणना > १०) {
    दर्शय("बहु")।
} अन्यथा {
    दर्शय("अल्प")।
}

चर इ = ०।
यावत् (इ < ५) { इ = इ + १। }        # while

प्रत्येकम् (अक्षरम् : सूची) {           # for … of
    दर्शय(अक्षरम्)।
}

Objects, arrays, strings

An object literal is कोष{ … } (kośa, "treasury"). Arrays use [ … ]. Member access and the standard-library methods read in Sanskrit — .दीर्घता is .length, .योजय(x) is .push(x).

नियत कविः = कोष{ नाम: "कालिदासः", वर्षम्: ४०० }।
दर्शय(कविः.नाम)।                       # कालिदासः

नियत सूची = ["अ", "आ", "इ"]।
सूची.योजय("ई")।                       # push  → .दीर्घता is now 4
नियत बडे = सूची.गालय(कार्य(x){ फलम् x != "अ"। })।   # filter

Strings interpolate with अन्तर्न्यासः — write पाठ"…{expr}…":

नियत नाम = "सीता"।
दर्शय(पाठ"नमस्ते {नाम}, स्वागतम्।")।   # नमस्ते सीता, स्वागतम्।

Operators & sugar

Arithmetic and comparison use the familiar symbols (+ - * / %, == != < > <= >=, && ||). Sweeteners include compound assignment (+=), the ternary, and the nullish default operator अथवा ("or else").

गणना += १।
नियत स्थितिः = सिद्धम् ? "✓" : "…"।
नियत पत्तनम् = पर्यावरण("PORT") अथवा "8080"।   # default when null

Branch & pattern match

विकल्प is a switch with no fall-through, and it doubles as a pattern matcher over object and array shapes.

विकल्प (आकृतिः) {
    स्थिति कोष{ प्रकारः: "वृत्तम्", त्रिज्या: र }:
        फलम् ३.१४ * र * र।
    स्थिति कोष{ प्रकारः: "चतुरः", भुजा: भ }:
        फलम् भ * भ।
    अन्यथा:
        फलम् ०।
}

Destructuring

नियत कोष{ नाम, वर्षम् } = कविः।         # object destructuring
नियत [प्रथमम्, द्वितीयम्] = सूची।      # array destructuring

Gradual, erasable types

Type annotations are optional and erased after checking — they add zero runtime cost, exactly like TypeScript. Base types are सङ्ख्या (number), अक्षर (string), तथ्य (boolean), वस्तु (object), रिक्त (void), and किमपि (any). Composite forms cover arrays, object shapes, and function types.

नियत वयः: सङ्ख्या = ३०।
नियत नामानि: गण<अक्षर> = ["अ", "आ"]।     # array of strings
नियत कविः: { नाम: अक्षर, वर्षम्: सङ्ख्या } = कोष{ नाम: "क", वर्षम्: ४०० }।

कार्य योग (अ: सङ्ख्या, ब: सङ्ख्या): सङ्ख्या {   # typed params + return
    फलम् अ + ब।
}
Check without running

node src/cli.js check file.deva type-checks a program and reports mismatches like प्रकारभेदः: गण<अक्षर> ← गण<सङ्ख्या> — with no code executed.

Modules

निर्यात exports, आयात { … } आ "path" imports. Paths beginning with std/ resolve to the shipped standard library from anywhere. Imports support aliasing (रूपेण) and re-export barrels.

# गणितागारम्.deva
निर्यात कार्य वर्गः (न) { फलम् न * न। }

# मुख्य.deva
आयात { वर्गः } आ "./गणितागारम्"।
आयात { मार्गकः } आ "std/मार्गकः"।    # the standard library
दर्शय(वर्गः(७))।

Async & await

असमकालिक कार्य is an async function; प्रतीक्षा awaits. Promise methods chain in Sanskrit: .ततः (then), .दोषे (catch), .अन्ततः (finally).

असमकालिक कार्य आनयतु () {
    नियत परि = प्रतीक्षा जाल.आनयप्रदत्त("/api/सूची")।
    यदि (परि.सफल) { दर्शय(परि.मूल्यम्.प्रदत्तम्)। }
}

The Result model — errors as values

Devabhāṣā has no exceptions. A fallible operation returns a परिणाम (Result): साधितम्(value) on success, विफलम्(error) on failure. The shape is { सफल, मूल्यम्, दोषः }.

कार्य भागः (अ, ब) {
    यदि (ब == ०) { फलम् विफलम्("शून्येन भागः")। }
    फलम् साधितम्(अ / ब)।
}

उद्धृ (uddhṛ, "extract") unwraps a Result or propagates its failure to the enclosing function — Rust's ?. अथवा supplies a fallback value.

असमकालिक कार्य मुख्यम् () {
    नियत मूल्यम् = उद्धृ भागः(१०, २)।       # unwrap, or return the विफलम्
    नियत सुरक्षितम् = भागः(१, ०) अथवा ०।    # default on failure
    फलम् साधितम्(मूल्यम्)।
}
Why it matters

Every fallible edge — file I/O, network, JSON parsing, database access, schema validation — returns the same परिणाम. Failures are ordinary values you thread through the program with उद्धृ, never control-flow that escapes past a missing try.
