Standard Library

Method names read as Sanskrit and lower to the JavaScript you'd expect — .दीर्घता is .length, पद.उच्च() is .toUpperCase(). The higher-level modules under std/ are themselves written in Devabhāṣā.

Math (गणित)

गणित is Math — with classically-named trigonometry. गणित.ज्या (jyā, the historical Sanskrit term that gave English "sine") is Math.sin; गणित.कोटिज्या is cosine.

The web layer

Devabhāṣā builds DOM with a Pāṇinian construction verb (रचय / अङ्गम्), styles it with a Sanskrit CSS vocabulary (रूप), and drives it with fine-grained reactivity.

SanskritDoes
रचय tag वाक्यम् text रूप {…} {children}create an element (text, styles, children — all optional)
योजय(node[, target])mount / append to the DOM
श्रोता(node, ev, fn)addEventListener
भाव x = …a reactive state cell (reassign to update)
प्रभाव(fn)an effect that re-runs when the cells it reads change
बन्ध(fn)a text node bound to a reactive thunk
आवली(…)keyed list reconciliation
रूप { वर्णः: …, पृष्ठभूमिः: … }inline styles in Sanskrit (color, spacing, layout)

Colors and layout keywords are Sanskrit too: रक्तः (red), नीलः (navy), श्वेतः (white), प्रवाहः (flex), मध्ये (space-between). Unknown property names and values pass through unchanged, so raw CSS still works.

File & network I/O

Every I/O operation is असमकालिक and returns a परिणाम — a failed read or fetch is a value, never a throw. The program is bound to an __IO interface, so the same source runs on the Node backend or a browser shim.

SanskritDoes
सञ्चिका.पठ(पथ) / .लिख(पथ, द)read / write a file
सञ्चिका.पठप्रदत्त / .लिखप्रदत्तread + parse / serialize + write JSON
सञ्चिका.विद्यते(पथ) / .निष्कासय(पथ)exists / remove
जाल.आनय(url[, opts])fetch → { स्थितिः, पाठः, सफलम् }
जाल.आनयप्रदत्त(url[, opts])fetch + parse JSON → { स्थितिः, प्रदत्तम्, सफलम् }

The std/ modules

Import these with आयात { … } आ "std/<name>". Each is written in Devabhāṣā.

ModuleProvides
std/आकृतिruntime schemas — validate untrusted input against a shape → a path-qualified परिणाम
std/मार्गकःa Koa-style router: method + path with :params and middleware
std/कोष्ठागारa JSON-file document store (dependency-free, browser-portable)
std/सञ्चयःthe same store API, backed by SQLite (durable, queryable)
std/प्रमाणम्auth — signed tokens (पत्रम्) & salted password hashing (गूढपदम्)
std/सत्रsigned-cookie sessions middleware
std/कुकीcookie parsing & Set-Cookie building
std/जालमध्यगाःstatic-file serving (स्थैतिक) and CORS (कोर्स)
std/प्रलेखstructured logging — one JSON line per event
std/परीक्षाa small test framework (अपेक्ष / समम्)

The persistence, validation, and server modules come together in the Backend & Full-stack guide.
