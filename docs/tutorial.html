Tutorials

Six short steps from a first program to a full-stack app — each one runnable with the bundled compiler. No dependencies to install.

0 · Setup

You need Node.js 18+ (22.5+ for the SQLite chapters). Clone the repo — there is nothing to npm install, because the compiler is dependency-free.

git clone https://github.com/pariharshyamu/devabhasha
cd devabhasha
node src/cli.js run examples/namaste.deva     # smoke test

The CLI has four verbs: run (compile + execute on Node), build (emit a .js file), check (type-check only), and serve (a live dev server for browser programs).

1 · Hello, Devabhāṣā

Create अभिवादनम्.deva:

# अभिवादनम्.deva
कार्य अभिवादनम् (नाम) {
    फलम् पाठ"नमस्ते, {नाम}!"।
}

दर्शय(अभिवादनम्("जगत्"))।

Run it:

node src/cli.js run अभिवादनम्.deva
# → नमस्ते, जगत्!

Want to see the JavaScript? node src/cli.js build अभिवादनम्.deva writes अभिवादनम्.js next to it — readable output you can read and debug.

2 · A CLI with Results

Devabhāṣā has no exceptions. Let's write a divider that returns a परिणाम and thread it with उद्धृ. Create गणकः.deva:

# गणकः.deva — a tiny calculator over the Result model
कार्य भागः (अ, ब) {
    यदि (ब == ०) { फलम् विफलम्("शून्येन भागः न सम्भवति")। }
    फलम् साधितम्(अ / ब)।
}

कार्य मुख्यम् () {
    नियत क = उद्धृ भागः(१००, ४)।      # 25, or bail out with the विफलम्
    नियत ख = भागः(१, ०) अथवा ०।       # default 0 on failure
    फलम् साधितम्(पाठ"क={क}, ख={ख}")।
}

नियत परि = मुख्यम्()।
यदि (परि.सफल) { दर्शय(परि.मूल्यम्)। } अन्यथा { दर्शय("दोषः: " + परि.दोषः)। }
node src/cli.js run गणकः.deva
# → क=25, ख=0
The idea

उद्धृ unwraps a success or returns the failure from the enclosing function — so the happy path reads top-to-bottom, and errors are values, never thrown.

3 · Add types

Annotations are optional and erased after checking. Give the calculator typed signatures:

कार्य भागः (अ: सङ्ख्या, ब: सङ्ख्या): परिणाम {
    यदि (ब == ०) { फलम् विफलम्("शून्येन भागः")। }
    फलम् साधितम्(अ / ब)।
}

Now type-check without running:

node src/cli.js check गणकः.deva
# ✓ प्रकारपरीक्षा सफला (no type issues)

Pass a string where a number is expected and check reports a प्रकारभेदः (type mismatch) with the offending location — all before any code runs.

4 · A reactive web page

The web layer builds DOM in Sanskrit, styles it with रूप, and reacts with भाव (a reactive cell) + प्रभाव (an effect that re-runs when its cells change).

# गणना.deva — a click counter
भाव गणना = ०।

चर शीर्षः = रचय शीर्षः वाक्यम् बन्ध(कार्य(){ फलम् पाठ"गणना: {गणना}"। })।
चर पटः = रचय पटः वाक्यम् "वर्धय" रूप { पृष्ठभूमिः: नीलः, वर्णः: श्वेतः }।
श्रोता(पटः, "click", कार्य(){ गणना = गणना + १। })।   # reassign → the heading re-renders

योजय(शीर्षः)। योजय(पटः)।

Preview it live with the dev server (hot-reloads on save), then open the printed URL:

node src/cli.js serve गणना.deva

See the web layer reference for case-role construction, styling words, keyed lists, and components.

5 · A backend service

A real HTTP service on the standard library: मार्गकः routes, आकृति validates, and सञ्चयः persists to SQLite. Create सेवा.deva:

# सेवा.deva
आयात { मार्गकः } आ "std/मार्गकः"।
आयात { सञ्चयः } आ "std/सञ्चयः"।
आयात { वस्तु, अक्षर, परीक्ष्य } आ "std/आकृति"।

नियत आकृतिः = वस्तु(कोष{ शीर्षकम्: अक्षर, लेखकः: अक्षर })।

असमकालिक कार्य मुख्यम् () {
    नियत आधार = उद्धृ दत्ताधारः.उद्घाटय("ग्रन्थाः.db")।
    नियत कोशः = सञ्चयः(आधार, "ग्रन्थाः", कार्य(र){ फलम् परीक्ष्य(आकृतिः, र)। })।

    नियत ऐप = मार्गकः()।
    ऐप.प्राप्("/ग्रन्थाः", असमकालिक कार्य(अ, प्र){
        नियत सर्वे = प्रतीक्षा कोशः.आनय()।
        प्र.प्रेषय_जेसन(सर्वे.मूल्यम्)।
    })।
    ऐप.स्थापय("/ग्रन्थाः", असमकालिक कार्य(अ, प्र){
        नियत देहः = प्रतीक्षा अ.देहम्_जेसन()।
        नियत योग = प्रतीक्षा कोशः.योजय(देहः.मूल्यम्)।   # आकृति validates here
        यदि (योग.सफल) { प्र.स्थिति(२०१).प्रेषय_जेसन(योग.मूल्यम्)। }
        अन्यथा { प्र.स्थिति(४२२).प्रेषय_जेसन(कोष{ दोषः: योग.दोषः })। }
    })।
    ऐप.चालय(८०८०)।
}
मुख्यम्()।
node src/cli.js run सेवा.deva
# then, in another terminal:
curl -X POST localhost:8080/ग्रन्थाः -d '{"शीर्षकम्":"गीता","लेखकः":"व्यासः"}'
curl localhost:8080/ग्रन्थाः

A malformed body comes back as a 422 with a path-qualified reason — the schema is the door, and nothing invalid reaches the database. Restart the process and the records are still there.

6 · A full-stack app

The repository ships a complete example — a book catalog whose frontend and backend are both Devabhāṣā — at examples/पूर्णस्तर/:

node examples/पूर्णस्तर/चालय.mjs
# → open http://localhost:8100
  • A reactive frontend (अग्रिम.deva) fetches the list, renders it with प्रभाव, adds books by POST, deletes with a ✕, and shows the backend's validation error inline.
  • A backend (पश्चिम.deva) runs मार्गकः + सञ्चयः (SQLite) + आकृति, and serves the compiled frontend at /.

Read the walkthrough in the Backend & Full-stack guide.
