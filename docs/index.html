A Sanskrit programming language

Write programs in संस्कृतम्.
Ship readable JavaScript.

Devabhāṣā is a small, dependency-free language whose source is written in Devanagari and transpiles to clean JavaScript — with a reactive web layer, a no-exceptions error model, gradual erasable types, a router, SQLite persistence, and a compiler that compiles itself.

# अभिवादनम् — a greeting (abhivādanam)
कार्य अभिवादनम् (नाम) {
    फलम् "नमस्ते, " + नाम।
}

दर्शय(अभिवादनम्("जगत्"))।   # → नमस्ते, जगत्

compiles to function abhivaadanam(naama){ return "नमस्ते, " + naama; }

📜

Sanskrit, all the way down

Keywords, the standard library, the router, even the compiler are written in Devanagari. Identifiers transliterate to stable ASCII, so the output JavaScript stays portable.

⚛️

A reactive web layer

Build DOM with Pāṇinian case roles, style it in Sanskrit (रूप), and drive it with fine-grained reactivity (भाव/प्रभाव) — no virtual DOM, no framework.

🛡️

Errors as values

No exceptions. Fallible operations return a परिणाम (Result); उद्धृ propagates failures like Rust's ?, and अथवा supplies defaults.

🔤

Gradual, erasable types

Optional type annotations (: सङ्ख्या, गण<अक्षर>, object shapes) are checked, then erased — zero runtime cost, TypeScript-style.

🗄️

A real backend story

An HTTP server, a Koa-style router, schema validation, signed-cookie sessions, and SQLite persistence — an end-to-end backend with no external dependencies.

♻️

Self-hosting

The lexer, parser, and codegen are written in Devabhāṣā and reach a byte-for-byte fixpoint — the compiler compiling the compiler produces itself.

Both ends, one language

A reactive Sanskrit frontend talks over real HTTP to a Sanskrit backend that routes, validates, and persists to a database. The परिणाम error model and JSON shapes are shared across the wire — the same सफल/मूल्यम्/दोषः on both sides.

Requirements

Devabhāṣā is dependency-free and runs on Node.js 18+. The optional SQLite persistence (दत्ताधारः/सञ्चयः) uses the built-in node:sqlite module, which needs Node.js 22.5+. In the browser it degrades to a clear failure Result.
