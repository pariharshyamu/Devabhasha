Backend & Full-stack

Devabhāṣā has an end-to-end backend story in one Sanskrit source, with no external dependencies: validate at the edge → persist → configure and log → authenticate → serve — all threaded through the परिणाम error model.

The backend arc

Five capabilities, each a standard-library module written in Devabhāṣā:

ConcernModule
Typed boundariesstd/आकृति — runtime schema validation
Persistencestd/कोष्ठागार (JSON file) · std/सञ्चयः (SQLite)
Service essentialsकाल (time) · पर्यावरण (env) · std/प्रलेख (logs)
Identity & securityगुप्ति (crypto) · std/प्रमाणम् (auth) · std/सत्र (sessions)
HTTP maturitystd/मार्गकः (router) · std/जालमध्यगाः (static, CORS) · std/कुकी

Server & router

सेवक is the minimal HTTP primitive; std/मार्गकः turns it into a clean router with method + path dispatch, :param capture, and Koa-style middleware. The request exposes मार्गः (path), रीतिः (method), शीर्षाणि (headers), प्राचलाः (path params), and देहम्_जेसन() (parse the body → परिणाम). The response has स्थिति(n), शीर्षम्(k,v), लेखय(text), and प्रेषय_जेसन(obj).

आयात { मार्गकः } आ "std/मार्गकः"।
नियत ऐप = मार्गकः()।

ऐप.प्राप्("/ग्रन्थाः/:अङ्कः", कार्य(अ, प्र){
    प्र.प्रेषय_जेसन(कोष{ अङ्कः: अ.प्राचलाः["अङ्कः"] })।
})।
ऐप.स्थापय("/ग्रन्थाः", कार्य(अ, प्र){ /* POST */ })।
ऐप.निष्कासय("/ग्रन्थाः/:अङ्कः", कार्य(अ, प्र){ /* DELETE */ })।
ऐप.उपयुज्(/* middleware */)।
ऐप.चालय(८०८०)।

The typed boundary — आकृति

The erased प्रकार types can't guard untrusted input (a request body, parsed JSON, an env var). std/आकृति mirrors the shape at runtime: परीक्ष्य(schema, value) returns the value or a path-qualified विफलम् like अङ्काः[1]: अपेक्षितम् सङ्ख्या, प्राप्तम् अक्षर. It pairs with उद्धृ.

आयात { वस्तु, अक्षर, सङ्ख्या, परीक्ष्य } आ "std/आकृति"।
नियत आकृतिः = वस्तु(कोष{ शीर्षकम्: अक्षर, वर्षम्: सङ्ख्या })।
नियत देहः = उद्धृ परीक्ष्य(आकृतिः, कच्चम्)।   # a typed, Result-carrying value

Persistence & SQLite

Two backends behind one API — योजय/आनय/एकम्/ अन्वेषय/परिवर्तय/निष्कासय/गणना, every method असमकालिक and परिणाम-returning:

  • std/कोष्ठागार — a JSON-file document store. Dependency-free and browser-portable; fine for small data.
  • std/सञ्चयः — the durable v2 on SQLite: ACID writes, no full-file rewrites, and raw SQL on the same handle. Needs Node 22.5+ (node:sqlite).
आयात { सञ्चयः } आ "std/सञ्चयः"।
नियत आधार = उद्धृ दत्ताधारः.उद्घाटय("ग्रन्थालयः.db")।
नियत कोशः = सञ्चयः(आधार, "ग्रन्थाः")।             # optional validator as 3rd arg
नियत लेखः = उद्धृ प्रतीक्षा कोशः.योजय(कोष{ शीर्षकम्: "गीता" })।   # ACID insert → record + अङ्कः
नियत सर्वे = उद्धृ प्रतीक्षा कोशः.आनय()।                          # every record → गण
The trade

कोष्ठागार stays dependency-free and runs anywhere; सञ्चयः gains real queries and durability at the cost of node:sqlite. Same API — pick per deployment.

Raw SQL — दत्ताधारः

Underneath the store is दत्ताधारः (dattādhāra, a "data-base" calque). Its handle methods each return a परिणाम, and bound params are passed as a गण: आदेश (exec), सर्वे (all rows), प्रथमा (one row), चालय (run → changes/last-id), पिधा (close).

नियत आधार = उद्धृ दत्ताधारः.उद्घाटय("काव्यम्.db")।
उद्धृ आधार.आदेश("CREATE TABLE कवयः(नाम TEXT, वर्षम् INTEGER)")।
उद्धृ आधार.चालय("INSERT INTO कवयः VALUES(?,?)", ["कालिदासः", ४५०])।
नियत पुराणाः = उद्धृ आधार.सर्वे("SELECT नाम FROM कवयः WHERE वर्षम् < ?", [६००])।

Identity & sessions

गुप्ति gives the crypto primitives (SHA-256, HMAC, scrypt, secure random, base64url). std/प्रमाणम् builds on them: पत्रम् issues and verifies tamper-evident signed tokens, and गूढपदम् does salted scrypt password hashing with a constant-time check. std/सत्र ties those to cookies as signed-cookie sessions — the session is a token in a cookie, so it needs no server-side store.

आयात { गूढपदम् } आ "std/प्रमाणम्"।
नियत गूढम् = गूढपदम्.रचय("मन्त्रः")।              # hash at signup
यदि (गूढपदम्.मेलय(दत्तम्, गूढम्)) { /* login ok */ }   # constant-time verify

Middleware & config

Middleware runs on every request (Koa-style); one that responds without calling अग्रे() short-circuits before routing. That's what lets these compose:

  • std/जालमध्यगाः — स्थैतिक(dir) serves static assets; कोर्स(opts) sets CORS headers and answers preflight.
  • std/सत्र — सत्रमध्यगः(key) reads/verifies the session into अ.सत्रम् and adds प्र.सत्रस्थापय / प्र.सत्रनाशः.
  • पर्यावरण(नाम) अथवा "default" is config-with-a-default; std/प्रलेख emits one greppable JSON line per event.
ऐप.उपयुज्(कोर्स())।                     # CORS on every response
ऐप.उपयुज्(सत्रमध्यगः(कुञ्जी))।          # signed-cookie sessions
ऐप.उपयुज्(स्थैतिक("./सार्वजनिक"))।       # static files, else fall through

A full-stack app

The repository ships examples/पूर्णस्तर/ — a book catalog whose frontend and backend are both Devabhāṣā, sharing the language, the परिणाम model, and the JSON shapes over the wire.

RouteDoes
GET /serves the HTML page running the compiled frontend
GET /api/ग्रन्थाःthe book list as JSON (from the database)
POST /api/ग्रन्थाःvalidate (आकृति) + persist (सञ्चयः) → 201, or 422
DELETE /api/ग्रन्थाः/:अङ्कःdelete by id
  • Frontend (अग्रिम.deva) — reactive DOM (भाव/प्रभाव): fetches the list, renders it, POSTs new books, deletes with a ✕, and surfaces the backend's validation error inline.
  • Backend (पश्चिम.deva) — मार्गकः + सञ्चयः (SQLite) + आकृति; seeds two books, and the data survives a restart.
  • Harness (चालय.mjs) — compiles the frontend to a browser bundle and runs the backend that serves it.
node examples/पूर्णस्तर/चालय.mjs
# → open http://localhost:8100
One language, both ends

A reactive Sanskrit frontend talks over real HTTP to a Sanskrit backend that routes, validates, and persists to a database — end to end, no dependencies beyond node:sqlite.
